# 05. 그리디 (Greedy)

## 개념
매 순간 **지금 가장 좋아 보이는 선택**을 하고 되돌아보지 않는 전략. 이 국소 최적이 전체 최적으로 이어진다는 **증명(교환 논증)** 이 성립할 때만 정답.

## 언제 쓰나
- "최소 개수", "최대 이익", "회의/작업 최대한 많이"
- 보통 **정렬 후** 앞에서부터 탐욕 선택
- DP로도 풀리지만 그리디가 성립하면 더 빠름

## 사고 순서
1. 어떤 기준으로 정렬할지 정한다 (끝나는 시간? 무게? 비율?)
2. 앞에서부터 조건 맞으면 선택
3. **반례를 떠올려 본다** — 안 떠오르면 (+직관) 제출, 의심되면 증명 시도

## C++ 예시 — 회의실 배정(1931)
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n; cin >> n;
    vector<pair<int,int>> a(n);          // {끝나는 시간, 시작 시간}
    for (auto& [end, start] : a) cin >> start >> end;

    sort(a.begin(), a.end());            // 끝나는 시간 오름차순
    int cnt = 0, lastEnd = 0;
    for (auto& [end, start] : a) {
        if (start >= lastEnd) {          // 겹치지 않으면 선택
            cnt++;
            lastEnd = end;
        }
    }
    cout << cnt;
}
```

| 11399 | ATM | S3 | 정렬 후 누적합 |
| 1931 | 회의실 배정 | S1 | **끝나는 시간 기준** 정렬 |
| 11047 | 동전 0 | S4 | 큰 단위부터 (배수 조건) |
| 1541 | 잃어버린 괄호 | S2 | 빼기 묶기 |
| 2839 | 설탕 배달 | S4 | 그리디/DP 둘 다 가능 |
| 1700 | 멀티탭 스케줄링 | G1 | 어려운 그리디 |

## 함정
- **그리디는 항상 성립하지 않는다.** 동전 문제도 단위가 배수 관계가 아니면 그리디 실패 → DP 필요.
- "정렬 기준"이 핵심. 시작 시간으로 정렬하면 회의실 문제 틀린다 (끝나는 시간이 맞음).
- 확신이 없으면 작은 입력으로 완전탐색 결과와 비교해 검증.
