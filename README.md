# 알고리즘 스터디 (백준 / C++)

> **목적: 취업 코딩테스트 대비** (삼성 SW역량테스트 · 카카오/라인/쿠팡 공채형)
> 현재 레벨: **solved.ac Gold II** (핸들 `zxcas57`) · 강점 그래프/BFS · 보강 중 DP
> 진행 상황: [문제집.md](문제집.md) (티어별 큐·다음 문제) · [progress.md](progress.md) (일자별 기록·solved.ac 현황)

## 진행 방식

1. **`문제 생성해줘`** → 현재 레벨/트랙에 맞는 **실제 백준 문제** 1개 제시
   - Bronze~Silver 하위: 자료구조/입출력 골격 + `// TODO` 힌트 스캐폴드
   - Silver 상위~Gold: 문제 설명 + 예시만 (백지에서 실전)
2. 직접 풀어 **백준에 제출** → 결과 보고 (`맞았어` / `틀렸어` / `막혔어`)
3. **`채점해줘`** → 예제 통과가 아니라 코드 검토 + 독립 레퍼런스 대조/스트레스 테스트로 검증
4. 통과 확인되면 **`기록해줘`** → progress.md + 문제집.md + Notion 트래커 동시 갱신
5. 막히면 **`힌트`** 단계별 (정답 코드는 `정답 보여줘` 요청 시에만 — 풀이는 사용자가 직접 작성)

## 명령어 / 스킬

| 입력 | 동작 |
|------|------|
| `문제 생성해줘` | 다음 문제 1개 제시 (`문제 생성해줘 BFS` 처럼 주제 지정 가능) |
| `힌트` | 현재 문제 단계별 힌트 |
| `정답 보여줘` | 모범 풀이 + 해설 (채팅 설명, 파일엔 미기입) |
| `채점해줘` | 풀이 코드 정적 검토 + 레퍼런스 대조 검증 *(스킬: grade-problem)* |
| `기록해줘` | progress·문제집·Notion 3곳 동시 기록 *(스킬: record-problem)* |
| `진도` | progress.md 현황 확인 |

## 커리큘럼 / 트랙

취업 코테 우선순위로 **`DP` · `구현/시뮬`(삼성형) · `백트래킹/브루트포스`** 를 먼저,
이어서 `자료구조` · `이분탐색` · `그리디` · `정렬` · `문자열` · `탐색(BFS/DFS)`.

**살구 스터디 커리큘럼(1~8주차, 트랙 A~Z)** 문제를 [문제집.md](문제집.md)에 티어별로 통합해 관리한다.
풀이 순서·다음 문제(`← 다음` 마커)·완료 카운트는 모두 문제집.md가 단일 소스.

## 폴더 구조
```
문제집.md                # 티어별 진행 현황 + 다음 문제 (단일 소스)
progress.md              # 일자별 풀이 기록 + solved.ac 현황
problems/
  bronze/   B3_boj_2884.cpp  B1_boj_1546.cpp ...
  silver/   S1_boj_1149.cpp  S3_boj_2579.cpp ...
  gold/     G4_boj_12865.cpp G5_boj_7576.cpp ...
.claude/skills/          # grade-problem(채점) · record-problem(기록)
.vscode/                 # 빌드/IntelliSense 설정
```
**파일명 규칙: `<티어>_boj_<번호>.cpp`** (티어 접두어 대문자 `B1`~`B4`/`S1`~`S5`/`G1`~`G5`, ASCII)
> 한글 파일명은 g++/gdb에서 깨져 ASCII로 통일. 번호는 백준 문제 번호.

## 로컬 빌드 환경 (Windows / g++)
- 컴파일러: **MinGW-w64 g++ 16.1.0** (WinLibs, PATH 등록 완료)
- VS Code에서 `.cpp` 열고 **`Ctrl+Shift+B`** → 컴파일 후 같은 폴더 `input.txt`로 자동 실행
  - 예제를 `input.txt`에 넣어두면 출력이 터미널에 바로 나옴 → 백준 제출 전 검증
- 수동 컴파일: `g++ -O2 -std=gnu++17 -static -o a.exe <파일>.cpp`
  - ⚠️ `-static` 필수 (WinLibs 동적 DLL 의존성 회피, 단일 exe로 실행)
