# 06. 이분탐색 (Binary Search)

## 개념
**정렬된** 데이터에서 탐색 범위를 절반씩 줄여 O(log N)에 찾는다. 응용으로 "정답을 직접 찾는 대신 정답 후보를 이분"하는 **매개변수 탐색(parametric search)** 이 Gold의 핵심.

## 언제 쓰나
- 정렬된 배열에서 값 존재/개수/위치
- "최댓값을 최소화 / 최솟값을 최대화" → 매개변수 탐색 신호
- 답이 단조성(어떤 값 이상이면 항상 가능)을 가질 때

## C++ — STL 활용
```cpp
vector<int> a = {1,3,3,5,7};
sort(a.begin(), a.end());

bool exist = binary_search(a.begin(), a.end(), 3);          // 존재?
int lo = lower_bound(a.begin(), a.end(), 3) - a.begin();    // 3 이상 첫 위치
int hi = upper_bound(a.begin(), a.end(), 3) - a.begin();    // 3 초과 첫 위치
int cnt = hi - lo;                                          // 값 3의 개수
```

## C++ — 매개변수 탐색 골격
```cpp
// "조건을 만족하는 최댓값"을 찾는 형태
long long lo = 1, hi = MAX, ans = 0;
while (lo <= hi) {
    long long mid = (lo + hi) / 2;
    if (possible(mid)) {     // mid가 가능하면 더 키워본다
        ans = mid;
        lo = mid + 1;
    } else {
        hi = mid - 1;
    }
}
cout << ans;
```

## 함정
- **반드시 정렬 후** 탐색. lower/upper_bound는 정렬 안 됐으면 무의미.
- 매개변수 탐색은 `mid` 계산 시 **오버플로** 주의 → `long long`.
- `(lo+hi)/2`는 음수 범위에서 편향될 수 있음 (보통 PS는 양수라 무방).
- 경계 조건(`<=` vs `<`, `mid+1` vs `mid`)에서 무한루프 자주 발생 → 골격 외워두기.
