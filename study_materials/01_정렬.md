# 01. 정렬 (Sorting)

## 개념
데이터를 일정 기준으로 나열. 대부분의 문제는 **직접 정렬 알고리즘을 구현할 필요 없이** `std::sort`를 쓴다 (도입부, 평균 O(N log N)).

## 언제 쓰나
- "오름차순/내림차순으로 출력"
- 그리디·이분탐색의 **전처리** (정렬해두면 탐욕 선택/탐색이 쉬워짐)
- 좌표/구간/문자열을 특정 기준으로 줄 세우기

## C++ 템플릿
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    vector<int> a = {5, 2, 8, 1};

    sort(a.begin(), a.end());                 // 오름차순
    sort(a.begin(), a.end(), greater<int>()); // 내림차순

    // 사용자 정의 기준: x 오름차순, 같으면 y 오름차순
    vector<pair<int,int>> p;
    sort(p.begin(), p.end(), [](const pair<int,int>& a, const pair<int,int>& b){
        if (a.first != b.first) return a.first < b.first;
        return a.second < b.second;
    });
    // pair는 기본적으로 first→second 사전순이라 위 람다는 사실 sort(p.begin(),p.end())와 동일

    return 0;
}
```

## 시간복잡도
| 방법 | 복잡도 | 비고 |
|------|--------|------|
| `std::sort` | O(N log N) | introsort, 가장 무난 |
| 계수정렬 | O(N + K) | 값 범위 K 작을 때 (10989) |
| `std::stable_sort` | O(N log N) | 동일값 순서 유지 필요 시 |

## 대표 백준 문제
| 번호 | 제목 | 레벨 | 포인트 |
|------|------|------|--------|
| 2750 | 수 정렬하기 | B2 | sort 입문 |
| 2751 | 수 정렬하기 2 | S5 | N=100만, O(N log N) 필수 |
| 10989 | 수 정렬하기 3 | B1 | 계수정렬 (메모리 제한) |
| 11650 | 좌표 정렬하기 | S5 | pair 사용자정렬 |
| 1181 | 단어 정렬 | S5 | 다중 기준(길이→사전순) |
| 10814 | 나이순 정렬 | S5 | stable 개념 |

## 함정
- **비교함수는 "strict weak ordering"**: `<=` 쓰면 런타임 에러 가능. 항상 `<`(엄격) 기준으로.
- `sort`는 같은 값의 상대순서 보장 안 함 → 순서 유지 필요하면 `stable_sort`.
- N이 100만 이상 + 값 범위 큰데 출력까지 많으면 입출력 가속 필수.
