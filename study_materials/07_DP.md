# 07. DP (동적 계획법, Dynamic Programming)

## 개념
큰 문제를 **작은 부분문제로 나누고, 그 답을 저장(메모이제이션)** 해 중복 계산을 없앤다. 두 조건이 핵심:
- **최적 부분구조**: 부분문제의 최적해로 전체 최적해를 만든다
- **중복 부분문제**: 같은 부분문제가 여러 번 등장

## 언제 쓰나
- "경우의 수 / 최댓값 / 최솟값"인데 완전탐색은 시간 초과
- 선택을 반복하며 이전 결과에 의존 (계단, 배낭, LIS)

## 두 가지 방식
```cpp
// Top-down (재귀 + 메모)
int memo[1001];
int dp(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = dp(n-1) + dp(n-2);
}

// Bottom-up (반복 + 테이블)  ← 대체로 더 빠르고 안전
int d[1001];
d[0] = 0; d[1] = 1;
for (int i = 2; i <= n; i++) d[i] = d[i-1] + d[i-2];
```

## 푸는 절차
1. **상태 정의**: `dp[i]` 가 "무엇"을 뜻하는지 한 문장으로 (예: i번째까지의 최대 합)
2. **점화식**: `dp[i]` 를 더 작은 dp들로 표현
3. **초기값 / 정답 위치** 결정
4. 채우는 순서 (작은 것 → 큰 것)

## 배낭(0/1 knapsack) 골격
```cpp
// 무게 한도 W, 물건 i의 (무게 w, 가치 v)
int dp[100001];                       // dp[j] = 무게 j일 때 최대 가치
for (int i = 0; i < N; i++)
    for (int j = W; j >= w[i]; j--)   // 뒤에서 앞으로! (각 물건 1번)
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
cout << dp[W];
```

## 함정
- **상태 정의가 전부.** 정의가 모호하면 점화식이 안 나온다 — 종이에 먼저 적어라.
- 배낭에서 무게 루프를 **뒤에서 앞으로** 돌려야 같은 물건 중복 사용을 막는다.
- 초기값(불가능 상태) 처리 주의: 최댓값 문제면 `-INF`, 개수면 0.
- Top-down 재귀 깊이 큰 경우 스택 주의 → Bottom-up 선호.
