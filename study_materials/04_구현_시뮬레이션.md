# 04. 구현 · 시뮬레이션 (Implementation · Simulation)

## 개념
특별한 알고리즘보다 **문제에 적힌 규칙을 코드로 정확히 옮기는** 능력. 시뮬레이션은 시간/단계에 따라 상태가 변하는 과정을 그대로 따라가는 구현.

## 언제 쓰나
- "다음과 같은 규칙으로 N번 진행했을 때..."
- 격자 위 이동(상하좌우/대각선), 회전, 방향 전환
- 조건 분기가 많은 게임/로봇/날짜 계산

## C++ 핵심 도구
```cpp
// 1) 방향 배열 (상하좌우)
int dx[4] = {0, 0, 1, -1};
int dy[4] = {1, -1, 0, 0};

// 8방향
int ddx[8] = {-1,-1,-1,0,0,1,1,1};
int ddy[8] = {-1,0,1,-1,1,-1,0,1};

// 2) 방향 회전 (시계방향으로 90도): dir = (dir + 1) % 4
//    반시계: dir = (dir + 3) % 4

// 3) 격자 범위 체크 함수
bool inRange(int x, int y, int n, int m) {
    return 0 <= x && x < n && 0 <= y && y < m;
}
```

### 시뮬레이션 골격
```cpp
while (조건) {          // 단계마다
    // 1. 현재 상태 읽기
    // 2. 규칙에 따라 다음 상태 계산
    // 3. 상태 갱신
    // 4. 종료 조건 검사
}
```

## 시간복잡도
- 보통 O(단계 수 × 한 단계 비용). 제약을 보고 단순 시뮬이 통과 가능한지 먼저 가늠.

## 함정
- **문제를 끝까지, 두 번 읽어라.** 구현은 "안 읽어서" 틀리는 비율이 가장 높다.
- 방향/회전 규칙은 `dx,dy`와 `% 4`로 표준화하면 실수 급감.
- 1-indexed / 0-indexed 혼동 주의. 격자 경계 처리 빠뜨리지 말 것.
- 상태 변화는 **임시 버퍼에 계산 후 한꺼번에 반영**해야 할 때가 많다 (동시 갱신 문제).
